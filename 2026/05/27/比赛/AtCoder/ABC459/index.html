问题陈述

有 NNN 个单元格，从左到右排成一行。起初，任何一个单元格中都没有放置图块。
你会得到 QQQ 个查询，请按顺序处理它们。每个查询都是以下两种类型之一：

  • 1 x:在左起第 xxx 个单元格中放入 111 个图块。然后，如果每个单元格都至少有 111 个图块，则从每个单元格中移除 111 个图块。
  • 2 y:输出至少有 yyy 个图块的单元格数量。

分析：

题意大概就是俄罗斯方块。

  • 输入1和x时在x处堆一个方块，方块满一整排就会消除。
  • 输入2和x时询问高度不小于x的列数。

思路：

用aia_iai​记录第i列的高度，用sumjsum_jsumj​数组统计高度为j的列数。每次当sumj==Nsum_j==Nsumj​==N的时候，就说明高度为jjj的列数已经充满了，可以消掉，因此每次再留一个变量mnmnmn记录消除的次数。
当询问高度不小于x的列数时，由于我们消除掉了mnmnmn次，实际上就是问未消除时高度不小于x+mnx+mnx+mn的列数，因此输出sumx+mnsum_{x+mn}sumx+mn​即可。

！！！注意x+mnx+mnx+mn是可能越界的，因此数组要开的大一些，本人因此WA了一发…

问题陈述

给你一个由小写英文字母组成的字符串 SSS 。
请判断是否有可能重新排列 SSS 中的字符，使相邻的两个字符都不相同。
给你 TTT 个测试用例，请逐个求解。

思路：

我们统计出现最多次数的字符ccc，出现次数为sumcsum_csumc​，我们需要用其他字符作隔板，如果sumcsum_csumc​超过一半，我们就无法将所有的ccc隔开，输出No，具体来说就是：
设SSS长度为LLL，

  • 当sumc>⌊L+12⌋sum_c>\lfloor \frac{L+1}{2}\rfloorsumc​>⌊2L+1​⌋时，输出No。
  • 否则，输出Yes。

对于Yes的情况，考虑如何输出。
两种思路，先按照出现次数逆序排序后。

  1. 从出现最多的字符开始，先填入奇数位，再填入偶数位。
  2. 将二元组(sumi,c)(sum_i,c)(sumi​,c)加入大根堆中，每次取出两个，按照顺序填入后再放入大根堆中，不断进行。（赛时我采用的此方案）

对于本题，我一开始看到是树的处理，便有点儿畏缩了，但是实际上本题非常简单

问题陈述

有根树 TTT ，具有 NNN 个顶点，其中顶点编号为顶点 111 、顶点 222 、 …\ldots… 、顶点 NNN 。
顶点 111 是 TTT 的根，顶点 iii (2≤i≤N)(2 \leq i \leq N)(2≤i≤N) 的（直接）父级是 PiP_iPi​ 。

此外，顶点 iii (1≤i≤N)(1 \leq i \leq N)(1≤i≤N) 有 CiC_iCi​ 糖果。所有 (C1+C2+⋯+CN)(C_1 + C_2 + \cdots + C_N)(C1​+C2​+⋯+CN​) 糖果都可以相互区分。

高桥向 NNN 松鼠发出指令。具体来说，他向第 iii -th 松鼠 (1≤i≤N)(1 \leq i \leq N)(1≤i≤N) 发出了以下指令：

  • 从以顶点 iii 为根的子树中选择并收集 DiD_iDi​ 糖果。

不同的松鼠不能吃同样的糖果。
输出选择糖果的可能方式的数量（以 998244353998244353998244353 为模）。

在这里，即使最终选择的糖果组是相同的，如果选择它们的松鼠不同，它们也会被算作不同的方式。
如果不可能所有松鼠都按照指示带回糖果，则输出 000 。

分析：

由于松鼠需在iii为根的子树中选择DiD_iDi​个糖果，设iii为根的子树中剩余RiR_iRi​个糖果，则该松鼠有(RiDi)\binom{R_i}{D_i}(Di​Ri​​)，由于先处理根会影响子树的选择，我们需要从子树处理，而非从根开始排着处理。

思路：

通过dfs遍历子树，遍历到最深处时再处理答案，分为两种情况：

  • 若Ri<DiR_i<D_iRi​<Di​ ，则无法满足该松鼠，ans=0ans=0ans=0，终止dfs，输出即可。
  • 若Ri≥DiR_i\ge D_iRi​≥Di​ ，则ansansans乘以(RiDi)\binom{R_i}{D_i}(Di​Ri​​)，同时RiR_iRi​减少DiD_iDi​，去掉已经拿走的糖果。

对于RRR的处理，在dfs回溯时，将子节点的R′R'R′全加到RRR上即可。

问题陈述

给你一个长度为 NNN 的非负整数序列 A=(A1,A2,…,AN)A = (A_1, A_2, \ldots, A_N)A=(A1​,A2​,…,AN​) 。
你可以对 AAA 执行以下操作 0 次或更多次：

  • 选择一个有 1≤i≤N−11 \le i \le N - 11≤i≤N−1 的整数 iii ，将 AiA_iAi​ 减少 111 ，并将 Ai+1A_{i+1}Ai+1​ 增加 111 。

求使 AAA 严格递增所需的最小运算次数。
可以证明答案小于 2632^{63}263 。
给你 TTT 个测试用例，请逐个求解。

分析/思路：

处理本题问题的核心是：将严格递增转化为非递减。
通过将AiA_iAi​转化为Di=Ai−iD_i=A_i-iDi​=Ai​−i，可以同步转化以下条件：

  1. Ai<Ai+1→Di<=Di+1A_i<A_{i+1} \rightarrow D_i<=D_{i+1}Ai​<Ai+1​→Di​<=Di+1​
  2. Ai−1,Ai+1+1→Di−1,Di+1+1A_i-1,A_{i+1}+1 \rightarrow D_i-1,D_{i+1}+1Ai​−1,Ai+1​+1→Di​−1,Di+1​+1

因此我们就只需要让DDD更加“平均”即可。
那么怎么让DDD更加“平均”呢？
这里我采用归并思想：
假设我们遍历到DiD_iDi​处，iii前已经实现非递减（即尽可能平均），设前面的平均值为xxx。

  • 如果Di>=xD_i>=xDi​>=x，则满足条件，DiD_iDi​不需要动。
  • 如果Di<xD_i<xDi​<x，则x需要从前面“借”一些过来，我们将前面那一部分与 DiD_iDi​ 合并，那么这一部分的均值xxx会提升，而DiD_iDi​会下降。

这样操作会将数据分解为若干块，并且块的均值满足非递减，那么我们只需要分配块内的数据，让块内也满足非递减，就能构造出最优的D′D'D′
显然，第二种情况是可能连续合并的，这也符合情况：如果每个数据的欠的太多，赤字太大，就需要往前面借非常多。

在我们构造出D′D^{\prime}D′后，我们就需要考虑怎么得出最终的ansansans了。
这里考虑ansansans是怎么一步一步累积起来的。
对于一次操作Di−1,Di+1+1D_i-1,D_{i+1}+1Di​−1,Di+1​+1，ansansans会增加1。
如果最后构造出 Di′D^{\prime}_iDi′​ ，设 Xi=Di−Di′X_i=D_i-D^{\prime}_iXi​=Di​−Di′​ ，那么在第iii进行了XiX_iXi​次操作，将DiD_iDi​的数据往后移动了一格，那么ansansans会增加 XiX_iXi​ 。
但是实际上数据并不会只移动一格，而可能往后移动多次，假设从DiD_iDi​处借1个大小，向后移动到 Di+rD_{i+r}Di+r​ ，那么 ansansans 会增加 rrr ，而 DDD 中体现的则是， Di+1D_{i+1}Di+1​ 至 Di+r−1D_{i+r-1}Di+r−1​ 不变， DiD_iDi​ 减1， Di+rD_{i+r}Di+r​ 加1，那么这个 rrr 怎么来呢，我们取 DDD 的前缀和为PPP，发现对于这一变化 PiP_iPi​ 至 Pi+r−1P_{i+r-1}Pi+r−1​均减小1，总共减小rrr，而Pi+rP_{i+r}Pi+r​不变。
由此我们可以得出ans=∑1NPi−Pi′ans=\sum\limits_{1}^{N}P_i-P'_{i}ans=1∑N​Pi​−Pi′​。

由于DDD可能为负数，在构造块内数据的时候可能会出问题（由于C++的特性，余数可以为负）

复杂度O(T×N)O(T\times N)O(T×N)
